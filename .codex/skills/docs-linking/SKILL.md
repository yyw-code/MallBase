# 文档放置与导航规范

## 放置策略

| 文档范围 | 放置位置 |
|---|---|
| 跨模块架构、安装、部署、贡献与项目级说明 | `docs/` |
| 单一模块的设计、接口契约与测试说明 | 对应模块目录 |
| 测试基线与触发矩阵 | `docs/testing/` 或相应测试目录旁 |

文件名优先使用 kebab-case，文案风格与项目现有中文文档保持一致。

## 导航职责

- 根 `README.md` 只维护重要入口和面向首次访问者的精简导航，不要求列出全部文档。
- `docs/index.md` 负责 `docs/` 下可长期维护文档的完整分类导航。
- 模块专属文档由模块 README 或同目录入口维护；无需重复加入根 README。
- 新增、移动或删除文档时，只更新承担该层级导航职责的入口。

## 强制规则

1. 不在根 README 堆叠全部文档，也不维护与 `docs/index.md` 重复的完整清单。
2. 不保留指向不存在文件的链接。
3. 文档移动或删除时，必须搜索并更新仓库内引用。
4. 相对链接从当前文档位置计算，不能假设都从项目根目录起算。
5. 临时计划、测试报告和中间产物放入 `.codex/work/`，不进入正式文档导航。

## 自检清单

- [ ] 文档位置符合项目级或模块级边界。
- [ ] 对应层级的导航入口已更新。
- [ ] 仓库内不存在旧路径或失效链接。
- [ ] 根 README 仍保持精简，`docs/index.md` 承担完整导航。
