# 余额业务逻辑

本文面向产品、测试、客服、财务和开发人员，说明余额账户、余额流水、余额支付、余额退款、充值套餐和后台调整的规则。

## 核心概念

| 概念 | 说明 |
|------|------|
| 余额账户 | 用户当前余额聚合值 |
| 可用余额 | 可用于余额支付的金额 |
| 冻结金额 | 预留给冻结类业务，当前余额支付不使用 |
| 余额流水 | 每次余额变动的审计记录 |
| 余额支付 | 用户使用可用余额支付普通订单 |
| 余额退款 | 余额支付订单退款后退回余额 |
| 充值套餐 | 前台充值页展示的套餐配置 |

## 金额口径

余额底层按分存储：

| 层级 | 口径 |
|------|------|
| 数据库 | 整数分，例如 `1000` 表示 10.00 元 |
| 接口 | 字符串元，例如 `10.00` |
| 页面 | 元 |

金额计算不使用浮点数直接入库，避免小数误差。

## 余额账户

账户字段：

| 字段 | 含义 |
|------|------|
| `balance_cents` | 可用余额，单位分 |
| `frozen_cents` | 冻结金额，单位分 |
| `total_recharge_cents` | 累计充值或后台增加金额，单位分 |
| `total_consume_cents` | 累计消费或后台扣减金额，单位分 |

规则：

- 用户首次查询或调整余额时可自动创建钱包。
- 余额账户只保存聚合值。
- 所有余额变动必须写 `mb_user_wallet_log`。
- 不允许余额支付或后台扣减把可用余额扣成负数。

## 余额流水

流水字段：

| 字段 | 含义 |
|------|------|
| `biz_type` | 业务类型 |
| `biz_id` | 业务单号或主键 |
| `direction` | `income` 或 `expense` |
| `change_cents` | 变动金额，始终为正数 |
| `before_cents` | 变动前可用余额 |
| `after_cents` | 变动后可用余额 |
| `operator_type` | 系统、买家或管理员 |
| `operator_id` | 操作者 ID |
| `remark` | 备注 |

业务类型：

| 类型 | 含义 |
|------|------|
| `order_pay` | 订单余额支付 |
| `refund` | 售后退款退回余额 |
| `recharge` | 余额充值 |
| `admin_adjust` | 后台调整 |

当前代码中 `recharge` 类型已保留，完整充值自动入账能力仍需继续接入。

## 余额支付

余额支付流程：

1. 校验用户已登录。
2. 校验支付配置 `payment_balance_enabled` 已开启。
3. 加载当前用户的待支付订单。
4. 校验订单未超时、未支付、未关闭。
5. 事务内锁定订单和用户钱包。
6. 校验可用余额是否充足。
7. 扣减 `balance_cents`，增加 `total_consume_cents`。
8. 写余额流水 `order_pay`。
9. 写支付日志，支付方式为余额。
10. 推进订单状态到已支付。

阻止支付的情况：

- 用户未登录。
- 订单不存在或不属于当前用户。
- 订单不是待支付状态。
- 订单已超时。
- 余额支付未启用。
- 可用余额不足。
- 订单金额不合法。

## 余额退款

余额支付订单售后退款时，系统会退回余额。

流程：

1. 售后单进入可退款状态。
2. 判断原订单支付方式是余额支付。
3. 事务内锁定售后单、订单和钱包。
4. 增加用户 `balance_cents`。
5. 写余额流水 `refund`。
6. 推进售后单退款完成。

非余额支付订单不会走余额退款流程。

## 后台调整

后台调整余额是人工操作，必须填写原因。

规则：

- 管理员身份必须有效。
- 调整方向只能是增加或扣减。
- 调整金额必须大于 0。
- 单次调整金额不能超过 `999999.99` 元。
- 扣减时余额必须充足。
- 增加会同步增加 `total_recharge_cents`。
- 扣减会同步增加 `total_consume_cents`。
- 每次调整写流水 `admin_adjust`。

## 充值套餐

充值套餐字段：

| 字段 | 含义 |
|------|------|
| 支付金额 | 用户需支付金额 |
| 赠送金额 | 额外赠送金额 |
| 到账余额 | 支付金额 + 赠送金额 |
| 状态 | 是否在前台展示 |

当前前台充值页会展示启用套餐，并读取可用充值方式；提交时尚未形成完整的充值订单和支付回调闭环。

## 与其它模块的关系

| 模块 | 关系 |
|------|------|
| 支付配置 | 控制余额支付是否可用 |
| 订单 | 余额支付会推进订单为已支付 |
| 售后 | 余额支付订单退款退回余额 |
| 用户管理 | 后台查看余额流水和调整余额 |
| 客户端装修 | 个人中心余额入口由装修方案展示 |
