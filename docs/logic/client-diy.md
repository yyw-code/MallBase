# 客户端装修业务逻辑

本文面向产品、测试和开发人员，说明客户端装修模块的页面库、方案类型、系统默认、启用策略、快照、主题策略和运行时读取规则。

## 核心概念

| 概念 | 说明 |
|------|------|
| 页面库 | UniApp 页面路径和分类的统一来源 |
| 装修方案 | 首页、个人中心、底部导航、悬浮按钮的配置集合 |
| 系统默认方案 | 每种方案类型的兜底配置，不能修改或删除 |
| 当前启用方案 | 客户端运行时优先读取的方案 |
| 方案快照 | 每次启用方案时保存的发布快照 |
| 主题方案 | 颜色 token 集合，分浅色、深色和自定义 |
| 主题策略 | 用户是否可自选主题，以及后台默认主题模式 |

## 方案类型

| 类型 | 用途 | 主要字段 |
|------|------|----------|
| `home` | 首页装修 | `pageStyle`、`components`、`modules` |
| `profile` | 个人中心装修 | `pageStyle`、`modules` |
| `tabbar` | 底部导航装修 | `items`、`tabbar_mode` |
| `floating` | 全局悬浮按钮 | `enabled`、`items`、`hiddenPages`、`style` |

每种类型都应保留系统默认方案作为兜底。

## 系统默认方案

系统默认方案用于保证客户端始终有可用配置。

规则：

- 系统默认方案不能直接修改。
- 系统默认方案不能删除。
- 管理员可复制系统默认方案后编辑副本。
- 如果当前没有启用方案，客户端读取时回退到系统默认方案。
- 如果数据库中连系统默认方案也缺失，服务端使用代码内置 fallback。

## 启用策略

同一类型只允许一套当前启用方案。

启用流程：

1. 校验方案存在、未删除、状态启用。
2. 按方案类型规范化 schema。
3. 校验 schema 是否满足发布要求。
4. 事务内将同类型其它方案 `is_active` 置为 0。
5. 将当前方案 `is_active` 置为 1。
6. 写入 `mb_client_decoration_snapshot`。

删除规则：

- 系统默认方案不能删除。
- 当前启用方案不能删除。
- 只能删除未启用的非系统方案。
- 删除采用软删除，并把状态置为禁用。

## 运行时读取

客户端读取：

```text
GET /client/api/decorate/config
```

返回内容：

- 首页装修。
- 个人中心装修。
- 底部导航模式和 schema。
- 悬浮按钮。
- 主题配置。

读取优先级：

1. 当前启用方案。
2. 系统默认方案。
3. 代码内置 fallback。

返回前会做：

- schema 规范化。
- 图片资源归一化。
- 历史素材兼容。
- 预览字段清理。
- 部分业务开关联动过滤。

## 页面库

页面库保存 UniApp 页面元数据。

字段：

| 字段 | 含义 |
|------|------|
| `path` | 页面路径，规范为 `/` 开头 |
| `page_type` | `tab`、`page`、`subpackage` |
| `category` | 页面分类 |
| `package_root` | 分包 root |
| `need_login` | 是否需要登录 |
| `source` | `system`、`manual`、`auto` |

页面库用于：

- 首页组件跳转。
- 个人中心服务菜单。
- 底部导航。
- 悬浮按钮。
- 装修跳转目标选择器。

系统页面不能修改或删除。导入 UniApp `pages.json` 时，系统来源页面可被同步更新名称、分类和登录要求。

## 跳转目标选择器

装修跳转目标不只来自页面库，还聚合：

- 页面库页面。
- 商品。
- 商品分类。
- 品牌。
- 标签。
- 文章分类。
- 文章详情。

这样装修人员无需同时拥有所有业务菜单权限，也能在装修页面选择跳转目标。

## 主题策略

主题配置包含两层：

| 层级 | 说明 |
|------|------|
| 主题方案 | `mb_client_theme` 中的颜色 token |
| 主题策略 | 是否允许用户自选、后台默认模式、自定义主题 ID |

主题模式：

| 模式 | 说明 |
|------|------|
| `system` | 跟随系统 |
| `light` | 使用浅色主题 |
| `dark` | 使用深色主题 |
| `custom` | 使用指定自定义主题 |

客户端优先级：

1. 如果允许用户自选，优先使用用户主题偏好。
2. 如果不允许用户自选，使用后台主题策略。
3. 自定义模式必须有已发布自定义主题。
4. 主题表为空时，使用服务端 fallback 主题。

系统主题不能修改或删除；自定义主题发布前必须校验必要 token。

## 业务开关联动

装修配置会和业务开关联动。

当前明确联动：

| 业务 | 规则 |
|------|------|
| 积分 | `points_enabled` 关闭时，客户端配置会过滤个人中心积分入口 |
| 会员 | 个人中心前端会结合 `member_enabled` 和用户会员摘要决定是否展示会员入口 |
| 主题 | 用户是否可自选主题由主题策略控制 |

## 与其它模块的关系

| 模块 | 关系 |
|------|------|
| 上传素材 | 装修图片和图标需要可访问资源 |
| 商品 | 首页商品分组、商品跳转目标 |
| 内容 | 文章列表和文章详情跳转目标 |
| 会员 / 积分 / 余额 | 个人中心业务入口 |
| 用户设置 | 用户主题偏好保存和读取 |
