# 会员业务逻辑

本文面向产品、测试和开发人员，说明会员模块的账户模型、等级匹配、优惠计算、成长值累计和异常边界。

## 核心概念

| 概念 | 说明 |
|------|------|
| 会员等级 | 按成长值门槛配置的等级，包含等级名称和折扣百分比 |
| 用户会员账户 | 用户当前成长值、累计成长值、当前等级和等级来源 |
| 成长值流水 | 记录成长值变动和等级变更审计 |
| 会员权益模式 | 商品控制是否参与等级折扣或使用 SKU 会员价 |
| 会员优惠快照 | 下单时保存的会员等级和优惠金额 |
| 手动等级锁定 | 后台人工设置等级后，可阻止自动成长值升级覆盖 |

## 开关影响

当前会员能力由 `member_enabled` 统一控制。

| 开关 | 影响 |
|------|------|
| 会员总开关 | 关闭后客户端不展示会员入口，新订单不参与会员权益计算 |
| 成长值比例 | `member_growth_points_per_yuan`，每实付 1 元累计多少成长值，`0` 表示不累计 |

代码中等级、会员价和成长值能力都挂在会员总开关下。会员总开关关闭时，不再为新订单计算会员折扣，也不累计成长值。

## 会员账户

用户首次需要会员账户时，系统会自动创建账户。

账户字段：

| 字段 | 含义 |
|------|------|
| `growth_value` | 当前成长值 |
| `total_growth_value` | 累计获得成长值 |
| `level_id` | 当前会员等级 ID |
| `level_name` | 当前会员等级名称快照 |
| `level_source` | 等级来源，`auto` 或 `manual` |
| `level_lock_until` | 手动等级锁定到期时间，空表示长期锁定 |
| `level_remark` | 后台设置会员等级原因 |

## 等级匹配

自动等级匹配规则：

1. 只匹配启用中的会员等级。
2. 选择 `growth_min <= 当前成长值` 的最高门槛等级。
3. 门槛相同时，按排序和 ID 兜底。
4. 成长值增加后，只自动升级到更高等级。
5. 当前等级门槛被调高后，不会自动降级历史用户。
6. 手动锁定等级时，不会被自动升级覆盖。

示例：

| 当前成长值 | 可匹配等级 | 结果 |
|------------|------------|------|
| `0` | 普通会员 `0` | 普通会员 |
| `1200` | 普通会员 `0`、银卡会员 `1000` | 银卡会员 |
| `6000` | 普通会员 `0`、银卡会员 `1000`、金卡会员 `5000` | 金卡会员 |

## 手动等级

后台手动设置会员后，账户会记录：

- `level_source = manual`
- `level_lock_until`
- `level_remark`

锁定规则：

| 场景 | 结果 |
|------|------|
| `level_lock_until` 为空 | 长期锁定，不被自动升级覆盖 |
| `level_lock_until` 晚于当前时间 | 锁定期内不被自动升级覆盖 |
| `level_lock_until` 早于当前时间 | 视为过期，后续成长值变化可回到自动匹配逻辑 |
| 未选择锁定 | 记录为自动来源，后续成长值变化按自动规则处理 |

手动设置等级会写成长值流水 `admin_adjust`，变动成长值为 `0`，用于审计等级调整。

## 会员优惠计算

订单试算时，系统按订单项计算会员优惠。

商品会员权益模式：

| 模式 | 含义 |
|------|------|
| `global` | 使用会员等级折扣 |
| `disabled` | 不参与会员权益 |
| `level_discount` | 使用会员等级折扣 |
| `sku_price` | 使用 SKU 会员价 |

等级折扣规则：

- `100.00` 表示不打折。
- `95.00` 表示按原价 95% 计算。
- 折扣金额按订单项计算，再汇总到订单。
- 金额按分计算，避免浮点误差。

规格会员价规则：

- 商品模式必须是 `sku_price`。
- SKU 会员价必须大于 0。
- SKU 会员价低于普通销售价时才产生优惠。
- 优惠金额为 `(销售价 - 会员价) * 数量`。

计算示例：

| 场景 | 计算 |
|------|------|
| 等级折扣 `90.00`，商品 100 元买 1 件 | 优惠 10 元 |
| SKU 销售价 80 元，会员价 50 元，买 2 件 | 优惠 60 元 |
| 等级折扣 `100.00` | 优惠 0 元 |

## 成长值累计

订单完成后触发成长值累计。

成长值基数：

```text
min(订单实付金额, 订单商品总额 - 订单优惠金额)
```

再按 `member_growth_points_per_yuan` 计算成长值，结果向下取整。

示例：

| 项 | 值 |
|----|----|
| 商品总额 | 120.00 |
| 优惠金额 | 20.00 |
| 实付金额 | 100.00 |
| 成长值比例 | 1 |
| 最终成长值 | 100 |

幂等规则：

- 同一订单号只累计一次。
- 成长值流水使用 `biz_type = order_complete` 和订单号作为唯一业务键。
- 重复触发订单完成不会重复增加成长值。

## 订单快照

下单时，如果会员优惠金额大于 0，会写入订单会员优惠快照。

快照保存：

- 订单 ID。
- 订单号。
- 用户 ID。
- 下单时会员等级 ID。
- 下单时会员等级名称。
- 会员优惠金额。

等级名称、折扣或商品会员价后续变化时，历史订单不回算。

## 与其它模块的关系

| 模块 | 关系 |
|------|------|
| 商品 | 商品和 SKU 决定会员权益模式、会员价 |
| 订单 | 确认订单试算会员优惠，下单保存会员优惠快照 |
| 订单状态机 | 订单完成后触发成长值累计 |
| 客户端装修 | 会员总开关关闭时，个人中心会员入口不展示 |
| 用户管理 | 后台可查看和手动设置用户会员等级 |
