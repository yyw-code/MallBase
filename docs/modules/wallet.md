# 余额模块总览

余额模块负责用户余额账户、余额流水、余额支付、余额退款、充值套餐展示和后台余额调整。

## 适用人群

| 角色 | 推荐阅读 |
|------|----------|
| 运营 / 后台管理员 | [../operation/wallet.md](../operation/wallet.md) |
| 财务 / 客服 / 测试 | [../logic/wallet.md](../logic/wallet.md) |
| 开发 / 二次开发 | [../development/wallet.md](../development/wallet.md) |

## 功能范围

| 能力 | 说明 |
|------|------|
| 余额账户 | 每个用户一套余额账户，保存可用余额、冻结金额和累计金额 |
| 余额流水 | 所有余额变动写流水，便于客服排查和财务对账 |
| 余额支付 | 用户可用余额支付普通订单 |
| 余额退款 | 余额支付订单售后完成后退回用户余额 |
| 充值套餐 | 后台配置前台展示的充值套餐 |
| 后台调整 | 管理员可增加或扣减用户余额，并写入调整原因 |

## 当前边界

当前代码中，充值套餐已经有后台配置和前台展示；完整的“充值下单、支付回调、自动入账”闭环需要继续接入充值订单和支付回调。当前可用的余额入账方式主要是后台余额调整和余额退款。

## 主要后台入口

| 入口 | 作用 |
|------|------|
| 设置中心 -> 支付配置 | 开启余额支付、微信支付等支付渠道 |
| 营销 -> 充值套餐 | 配置充值金额、赠送金额、到账余额和展示图 |
| 用户 -> 用户管理 | 查看余额记录、后台调整用户余额 |
| 订单 -> 订单详情 | 查看余额支付方式和支付状态 |
| 售后 -> 售后单 | 余额支付订单完成退款后退回余额 |

## 主要前台入口

| 页面 | 路径 |
|------|------|
| 我的余额 | `/pages-sub/wallet/index` |
| 余额明细 | `/pages-sub/wallet/records` |
| 余额充值 | `/pages-sub/wallet/recharge` |
| 订单支付 | `/pages-sub/order/pay-result` 等订单支付链路 |

## 推荐配置流程

1. 在支付配置中开启余额支付。
2. 在用户管理中给测试用户增加余额。
3. 用测试用户创建普通订单。
4. 选择余额支付，确认订单支付成功。
5. 查看用户余额流水，应出现 `order_pay`。
6. 发起售后并完成退款，确认余额退回并出现 `refund` 流水。
7. 如需展示充值入口，配置充值套餐并在前台余额充值页确认展示。
