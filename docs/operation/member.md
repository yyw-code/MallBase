# 会员操作文档

本文面向后台管理员和运营人员，说明会员功能怎么配置、怎么验证、常见问题怎么处理。

## 适用场景

- 开启或关闭会员功能。
- 配置成长值比例和会员等级。
- 配置商品会员权益模式。
- 给用户手动设置会员等级。
- 验证会员折扣、成长值累计和自动升级。

如果需要理解计算规则，见 [../logic/member.md](../logic/member.md)。如果需要改代码，见 [../development/member.md](../development/member.md)。

## 后台入口

| 功能 | 入口 | 用途 |
|------|------|------|
| 会员配置 | `会员 -> 会员配置` | 会员总开关、成长值比例 |
| 会员等级 | `会员 -> 会员等级` | 配置等级名称、成长值门槛、等级折扣 |
| 用户会员 | `用户 -> 用户管理` | 查看会员信息、手动设置会员等级 |
| 商品会员权益 | `商品 -> 商品管理` | 配置商品是否参与会员折扣或使用规格会员价 |
| 订单会员快照 | 订单详情页 | 查看下单时的会员等级、会员优惠和成长值记录 |

## 配置前准备

配置前先确认：

1. 商品已经上架，并且至少有一个启用 SKU。
2. 订单下单、支付和完成流程可以正常使用。
3. 如果要验证规格会员价，商品已经开启多规格或至少有一个 SKU。
4. 如果要手动设置会员，后台角色拥有用户管理和设置会员权限。

## 推荐配置流程

### 1. 开启会员功能

进入 `会员 -> 会员配置`，确认：

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| 启用会员功能 | 开启 | 关闭后客户端不展示会员入口，新订单不参与会员权益计算 |
| 成长值比例 | 例如 `1` | 每实付 1 元累计多少成长值，`0` 表示不累计成长值 |

注意：会员总开关关闭后，历史订单会员快照不会回算或删除。

### 2. 配置会员等级

进入 `会员 -> 会员等级`，配置等级。

| 字段 | 说明 |
|------|------|
| 等级名称 | 前台和后台展示的等级名称 |
| 成长值门槛 | 用户成长值达到该门槛后可匹配该等级 |
| 等级折扣百分比 | `100.00` 表示不打折，`95.00` 表示按原价 95% 计算 |
| 排序 | 后台列表展示顺序 |
| 状态 | 禁用后不再作为自动匹配目标 |
| 备注 | 内部说明，不参与计算 |

建议至少保留一个成长值门槛为 `0` 的启用等级，作为新用户默认等级。

示例：

| 等级 | 成长值门槛 | 等级折扣 |
|------|------------|----------|
| 普通会员 | `0` | `100.00` |
| 银卡会员 | `1000` | `98.00` |
| 金卡会员 | `5000` | `95.00` |

### 3. 配置商品会员权益

进入商品编辑页，设置会员权益模式。

| 模式 | 含义 |
|------|------|
| 全局 | 使用会员等级折扣 |
| 关闭 | 该商品不参与会员权益 |
| 参与等级折扣 | 明确使用会员等级折扣 |
| 规格会员价 | 使用 SKU 上配置的会员价 |

如果没有复杂运营策略，建议先使用等级折扣，减少 SKU 级价格维护成本。

使用规格会员价时：

1. 商品会员权益选择 `规格会员价`。
2. 在 SKU 表格中填写会员价。
3. 会员价应低于普通销售价，否则不会产生有效优惠。

### 4. 手动设置用户会员等级

进入 `用户 -> 用户管理`，选择用户后设置会员等级。

适用场景：

- 客服补偿。
- 活动赠送会员等级。
- 线下导入用户等级。
- 临时保留某个等级，避免自动成长值规则立即覆盖。

操作要求：

- 必须选择启用中的会员等级。
- 必须填写调整原因。
- 可选择是否锁定等级。
- 锁定到期时间为空表示长期锁定。

手动设置会写入成长值流水，便于后续排查。

## 验证方法

### 验证等级折扣

1. 开启会员功能。
2. 配置一个低于 `100.00` 的会员等级折扣。
3. 给测试用户设置该会员等级。
4. 商品会员权益选择全局或参与等级折扣。
5. 前台进入确认订单页。
6. 确认会员优惠金额出现，订单应付金额减少。
7. 下单后打开订单详情，确认会员优惠快照存在。

### 验证规格会员价

1. 开启会员功能。
2. 商品会员权益选择规格会员价。
3. 在 SKU 上填写会员价。
4. 前台商品详情选择该 SKU。
5. 确认页面展示会员价或会员权益提示。
6. 确认订单页的会员优惠按销售价和会员价差额计算。

### 验证成长值累计和自动升级

1. 开启会员功能。
2. 设置成长值比例，例如 `1`。
3. 配置更高等级的成长值门槛。
4. 用测试用户完成一笔订单。
5. 查看用户详情，确认成长值增加。
6. 如果成长值达到更高等级门槛，确认会员等级自动升级。

### 验证手动锁定等级

1. 给测试用户手动设置较低等级，并选择锁定。
2. 完成一笔足够升级的订单。
3. 查看用户成长值，应正常增加。
4. 查看用户等级，应仍保持手动设置的等级。

## 常见问题

### 前台看不到会员入口

优先检查：

1. 会员总开关是否开启。
2. 客户端基础配置接口是否返回 `member_enabled = 1`。
3. 个人中心装修方案中是否有会员入口。
4. 当前用户是否已登录。

### 订单没有会员优惠

优先检查：

1. 会员总开关是否开启。
2. 用户是否有会员账户和有效等级。
3. 商品会员权益是否设置为关闭。
4. 等级折扣是否为 `100.00`。
5. SKU 会员价是否为空、为 0 或不低于销售价。

### 订单完成后没有成长值

优先检查：

1. 会员总开关是否开启。
2. 成长值比例是否大于 0。
3. 订单是否真正进入完成状态。
4. 订单实付金额是否大于 0。
5. 同一订单是否已经累计过成长值。

### 修改等级门槛后，老用户没有降级

当前自动逻辑只在成长值增加时评估升级，不会因为等级门槛调整自动降级。历史用户等级和历史订单快照以当时数据为准。
