# 积分操作文档

本文面向后台管理员和运营人员，说明积分功能怎么配置、怎么验证、常见问题怎么处理。

## 适用场景

- 开启或关闭积分功能。
- 配置消费返积分和订单积分抵扣。
- 配置积分商城商品。
- 处理积分兑换单。
- 查询用户积分流水和后台调整积分。

如果需要理解计算规则，见 [../logic/points.md](../logic/points.md)。如果需要改代码，见 [../development/points.md](../development/points.md)。

## 配置前准备

配置前先确认：

1. 商品已经上架，并且至少有一个启用 SKU。
2. 需要做积分商城的商品有可用库存。
3. 订单、支付、收货地址等基础流程可以正常使用。
4. 如果要验证冻结积分释放，测试环境可以执行 `php think points:release`。

## 推荐配置流程

### 1. 开启积分功能

在积分配置页确认：

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| 启用积分功能 | 开启 | 关闭后客户端和商品编辑不展示积分入口，并停止新发放和新抵扣 |
| 启用积分赠送 | 开启 | 关闭后订单完成不再产生新的赠送积分 |
| 积分冻结天数 | 例如 `7` | 订单完成后赠送积分先冻结，超过天数后释放 |
| 启用积分抵扣 | 按业务需要 | 开启后确认订单页可以用可用积分抵扣普通订单金额 |
| 积分抵扣比例 | 例如 `100` | 表示 100 积分抵扣 1 元 |
| 积分抵扣上限比例 | 例如 `50` | 表示单笔订单最多抵扣商品金额的 50% |

注意：关闭积分总开关后，历史返还、回收和释放仍会继续处理，避免历史数据中断。

### 2. 配置消费返积分规则

进入 `积分 -> 积分规则`，确认 `消费返积分` 规则。

| 字段 | 说明 |
|------|------|
| 场景 | 当前订单完成场景为 `order_complete` |
| 每元奖励 | 每消费 1 元奖励多少积分 |
| 单次上限 | 单笔订单最多奖励多少积分，`0` 表示不限制 |
| 状态 | 关闭后不再作为新订单的奖励规则 |

示例：

- 每元奖励：`1`
- 单次上限：`0`
- 含义：订单实付商品金额 128 元，理论奖励 128 积分。

### 3. 配置商品 / SKU 积分赠送模式

进入商品编辑页，商品和 SKU 可配置积分赠送模式。

| 模式 | 含义 |
|------|------|
| 全局 | 使用积分规则中的消费返积分 |
| 关闭 | 该商品或 SKU 不赠送积分 |
| 按金额 | 按该商品 / SKU 配置的每元积分计算 |
| 固定 | 按每件固定积分计算 |
| 规格单独配置 | 商品按 SKU 的积分配置计算 |

如果没有特殊运营策略，建议先使用全局模式，减少规则维护成本。

### 4. 配置订单积分抵扣

开启积分抵扣后，确认订单页会返回可用积分、可抵扣积分、实际使用积分和抵扣金额。

示例：

- 用户可用积分：`12000`
- 积分抵扣比例：`100` 积分抵 1 元
- 订单可抵扣商品金额：`100.00`
- 抵扣上限比例：`50`
- 最多抵扣金额：`50.00`
- 最多可用积分：`5000`

如果用户请求使用超过上限的积分，系统会按上限取可用值。

### 5. 配置积分商品

进入 `积分 -> 积分商品` 新增积分商品。

| 字段 | 说明 |
|------|------|
| 普通商品 | 选择已上架商品 |
| SKU | 选择具体 SKU，积分商品和 SKU 一一绑定 |
| 兑换积分 | 单件兑换需要消耗的积分，必须大于 0 |
| 兑换库存 | 积分商城可兑换库存 |
| 每人限兑 | 每个用户最多可兑换数量，`0` 表示不限制 |
| 状态 | 关闭后前台积分商城不展示 |

注意：

- 已有兑换记录的积分商品不能直接删除，可停用。
- 前台只展示启用、库存大于 0、普通商品已上架、SKU 启用的积分商品。

### 6. 处理积分兑换单

进入 `积分 -> 兑换订单` 处理兑换单。

| 状态 | 可执行操作 | 结果 |
|------|------------|------|
| 待发货 | 发货 | 状态变为已发货，记录物流或虚拟发货说明 |
| 待发货 | 关闭 | 返还积分，恢复积分商品库存和 SKU 库存 |
| 已发货 | 完成 | 状态变为已完成 |
| 已完成 | 无需处理 | 保留历史记录 |
| 已关闭 | 无需处理 | 保留关闭原因和操作记录 |

发货类型：

- 实物快递：填写物流公司和物流单号。
- 虚拟发货：填写发货说明。

### 7. 查看积分流水

积分流水可以按用户、业务类型查询。常见业务类型：

| 类型 | 含义 |
|------|------|
| `order_complete` | 订单奖励冻结 |
| `order_reward_release` | 积分释放 |
| `order_reward_release_frozen` | 冻结释放 |
| `refund` | 退款回收 |
| `refund_frozen` | 退款回收冻结 |
| `refund_debt` | 退款欠账 |
| `order_deduction` | 订单积分抵扣 |
| `order_deduction_return` | 订单取消返还 |
| `refund_deduction_return` | 退款返还抵扣积分 |
| `points_exchange` | 积分商品兑换 |
| `points_exchange_return` | 兑换关闭返还 |
| `admin_adjust` | 后台调整 |

### 8. 后台调整用户积分

进入用户管理，选择用户后查看积分记录或调整积分。

调整要求：

- 调整积分必须大于 0。
- 必须选择增加或扣减。
- 必须填写调整原因。
- 调整会写入积分流水。

不要直接修改数据库中的 `mb_user_points.balance_points`。

## 验证方法

### 验证消费返积分

1. 开启积分功能和积分赠送。
2. 设置消费返积分规则。
3. 用测试用户下单并完成订单。
4. 查看用户积分页，应出现冻结积分。
5. 执行 `php think points:release` 或等待计划任务。
6. 查看用户积分页，冻结积分应转为可用积分。

### 验证积分抵扣

1. 给测试用户增加可用积分。
2. 开启积分抵扣。
3. 进入确认订单页。
4. 勾选使用积分或输入使用积分。
5. 确认抵扣金额符合比例和上限。
6. 下单后查看积分流水，应出现 `order_deduction`。

### 验证积分商城兑换

1. 新增积分商品并启用。
2. 前台进入积分商城。
3. 选择商品、地址和兑换数量。
4. 提交兑换。
5. 查看用户积分减少、积分商品库存减少、兑换单为待发货。
6. 后台发货后状态变为已发货。
7. 后台完成后状态变为已完成。

### 验证关闭兑换单

1. 创建一笔待发货兑换单。
2. 后台关闭兑换单并填写原因。
3. 查看兑换单状态变为已关闭。
4. 查看用户积分已返还。
5. 查看积分商品库存和 SKU 库存已恢复。

## 常见问题

### 前台看不到积分入口

优先检查：

1. 积分总开关是否开启。
2. 客户端基础配置接口是否返回 `points_enabled = 1`。
3. 个人中心装修中是否有积分入口。
4. 当前客户端页面是否刷新到最新配置。

### 订单完成后没有积分

优先检查：

1. 积分总开关和积分赠送开关是否开启。
2. `消费返积分` 规则是否启用。
3. 商品或 SKU 是否关闭了积分赠送。
4. 订单是否真的进入完成状态。
5. 单次奖励上限是否限制为 0 或过低。

### 积分商品前台不展示

优先检查：

1. 积分商品状态是否启用。
2. 兑换库存是否大于 0。
3. 普通商品是否上架。
4. SKU 是否启用。
5. 积分总开关是否开启。

### 兑换失败提示库存不足

可能原因：

- 积分商品兑换库存不足。
- 关联 SKU 库存不足。
- 同一个用户已达到每人限兑数量。

### 关闭兑换单后积分没有返还

只有待发货兑换单可以关闭并返还积分。已发货或已完成兑换单不能通过关闭流程返还。
