# 客户端装修操作文档

本文面向后台管理员、运营和实施人员，说明客户端装修怎么配置、怎么发布、怎么验证、常见问题怎么处理。

## 适用场景

- 配置首页、个人中心、底部导航和悬浮按钮。
- 维护 UniApp 页面库，供装修跳转选择。
- 配置浅色、深色或自定义主题。
- 验证客户端运行时读取的装修配置。

如果需要理解方案回退、快照和主题策略，见 [../logic/client-diy.md](../logic/client-diy.md)。如果需要改代码，见 [../development/client-diy.md](../development/client-diy.md)。

## 后台入口

| 功能 | 入口 | 用途 |
|------|------|------|
| 页面库 | `客户端装修 -> 页面库` | 维护可跳转页面，导入 UniApp pages.json |
| 首页装修 | `客户端装修 -> 首页装修` | 配置搜索、轮播、导航、标题、商品分组等模块 |
| 个人中心装修 | `客户端装修 -> 个人中心装修` | 配置用户信息、会员、订单、余额、积分、服务菜单 |
| 底部导航 | `客户端装修 -> 底部导航` | 配置原生/自定义 TabBar、图标和页面链接 |
| 悬浮按钮 | `客户端装修 -> 悬浮按钮` | 配置全局悬浮入口、客服、购物车、首页等快捷操作 |
| 主题设置 | `客户端装修 -> 主题设置` | 管理主题、发布主题、配置主题策略 |

## 前台影响范围

| 配置 | 影响 |
|------|------|
| 首页装修 | `/pages/index/index` |
| 个人中心装修 | `/pages/profile/index` |
| 底部导航 | 主包 TabBar 和自定义 TabBar |
| 悬浮按钮 | 全局页面悬浮入口，排除隐藏页面 |
| 主题设置 | UniApp/H5 主题变量和用户主题偏好 |

## 配置前准备

配置前先确认：

1. H5/UniApp 客户端可以正常打开。
2. 页面库中已有要跳转的目标页面。
3. 上传配置可用，图片素材可访问。
4. 商品、分类、品牌、标签或文章等跳转目标已经启用。
5. 如果要展示会员、积分、余额入口，对应业务开关和页面可用。

## 页面库操作

页面库是装修跳转目标的统一来源。配置链接时优先从页面库选择，不建议手写路径。

页面字段：

| 字段 | 说明 |
|------|------|
| 页面名称 | 后台选择器展示名称 |
| 页面路径 | UniApp 页面路径，必须以 `/` 开头 |
| 页面类型 | `tab`、`page`、`subpackage` |
| 页面分类 | 基础、商品、内容、订单、售后、会员、营销、其它 |
| 分包 root | 分包页面所属 root |
| 是否需要登录 | 前台跳转时是否要求登录 |
| 来源 | `system`、`manual`、`auto` |
| 状态 | 禁用后不在选择器中展示 |

导入建议：

1. 从 UniApp `pages.json` 复制页面配置。
2. 在页面库执行导入。
3. 检查页面名称、分类、登录要求是否符合预期。
4. 系统页面不能修改或删除；自定义页面可维护。

## 装修方案操作

常见方案操作：

| 操作 | 说明 |
|------|------|
| 查看方案 | 按方案类型查看首页、个人中心、底部导航、悬浮按钮方案 |
| 新建方案 | 从空白方案开始配置 |
| 复制方案 | 推荐复制系统默认方案后编辑 |
| 编辑方案 | 调整模块、素材、跳转、样式 |
| 启用方案 | 当前类型只保留一个启用方案 |
| 删除方案 | 只能删除未启用的非系统方案 |

系统默认方案是兜底方案，不能直接修改或删除。需要调整时，应复制后编辑副本，再启用副本。

## 首页装修

首页常用模块：

| 模块 | 用途 |
|------|------|
| 搜索框 | 跳转搜索或商品列表 |
| 轮播图 | 配置图片、标题、跳转目标 |
| 导航宫格 | 配置多个分类或活动入口 |
| 标题栏 | 分区标题和更多跳转 |
| 商品分组 | 按推荐、新品、热销、分类、品牌、标签或指定商品展示 |

配置建议：

- 商品来源优先使用后台选择器，避免手写参数。
- 图片上传后确认客户端可访问完整 URL。
- 首页模块过多会影响首屏加载，建议先保证关键入口清晰。

## 个人中心装修

个人中心常用模块：

| 模块 | 用途 |
|------|------|
| 用户信息 | 头像、昵称、手机号等用户信息 |
| 会员入口 | 会员等级和成长值入口 |
| 我的订单 | 待付款、待发货、待收货、退款售后 |
| 我的余额 | 余额、余额明细、充值入口 |
| 我的积分 | 积分、积分记录、积分商城入口 |
| 我的服务 | 地址、设置、客服等服务菜单 |

业务开关联动：

- 会员总开关关闭时，客户端不展示会员入口。
- 积分总开关关闭时，客户端会过滤个人中心积分入口。
- 余额入口是否展示主要由装修方案和前台页面可用性决定。

## 底部导航

底部导航支持：

| 模式 | 说明 |
|------|------|
| 原生 TabBar | 使用 UniApp 原生 tabBar 页面 |
| 自定义 TabBar | 由客户端自定义组件渲染 |

配置注意：

- TabBar 页面路径应来自页面库中的 `tab` 类型页面。
- 图标和选中图标都应配置。
- 修改后同时检查 H5 和实际小程序/APP 表现。

## 悬浮按钮

悬浮按钮可配置：

- 是否启用。
- 展开或单按钮模式。
- 左右和底部偏移。
- 隐藏页面。
- 客服、购物车、首页或自定义页面入口。
- 图标、圆角、阴影、颜色等样式。

常见隐藏页面：

- 登录页。
- 用户协议页。
- 其它不适合出现悬浮入口的全屏流程页。

## 主题设置

主题设置包含：

| 配置 | 说明 |
|------|------|
| 主题列表 | 浅色、深色、自定义主题 |
| 发布状态 | 只有已发布主题会返回给客户端 |
| 允许用户自选主题 | 开启后用户偏好优先 |
| 管理员指定主题模式 | 跟随系统、浅色、深色、自定义 |
| 管理员指定自定义主题 ID | 仅自定义模式有效 |

自定义主题发布前必须包含必要主题变量，例如主色、背景色、文字色、边框色和价格色。

## 验证方法

### 验证装修方案生效

1. 复制系统默认方案。
2. 修改一个容易识别的标题、图片或入口。
3. 保存并启用方案。
4. 打开客户端实际页面。
5. 请求 `/client/api/decorate/config`，确认返回当前启用方案。
6. 确认页面渲染和后台预览一致。

### 验证页面跳转

1. 在页面库确认目标页面启用。
2. 在装修模块中选择目标页面。
3. 保存并启用方案。
4. 前台点击入口。
5. 确认跳转路径、登录要求和页面参数符合预期。

### 验证主题

1. 创建或复制自定义主题。
2. 填写完整主题变量。
3. 发布主题。
4. 在主题设置中选择自定义主题。
5. 前台刷新页面，确认颜色变量生效。
6. 如果允许用户自选主题，切换用户主题偏好后再次验证。

## 常见问题

### 后台改了方案，前台没变化

优先检查：

1. 是否只是保存，未启用方案。
2. 当前方案状态是否启用。
3. 客户端是否请求到最新 `/client/api/decorate/config`。
4. 浏览器或小程序是否有本地缓存。
5. 是否修改了系统默认方案但实际启用的是复制方案。

### 页面跳转失败

优先检查：

1. 页面路径是否以 `/` 开头。
2. 页面是否真实存在于 UniApp `pages.json`。
3. 分包路径是否正确。
4. 页面是否需要登录，当前用户是否已登录。
5. 是否手写了错误 query 参数。

### 系统默认方案不能编辑

这是正常限制。系统默认方案用于兜底，不能修改或删除。请复制后编辑副本。

### 积分入口配置了但前台不展示

优先检查积分总开关。积分关闭时，客户端读取装修配置会过滤个人中心积分入口。

### 主题发布失败

优先检查主题变量是否完整，至少需要主色、背景色、文字色、边框色和价格色等关键 token。
