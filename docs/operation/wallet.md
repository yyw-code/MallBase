# 余额操作文档

本文面向后台管理员、客服、财务和运营人员，说明余额功能怎么配置、怎么验证、常见问题怎么处理。

## 适用场景

- 开启或关闭余额支付。
- 给用户增加或扣减余额。
- 查询用户余额流水。
- 配置前台展示的充值套餐。
- 验证余额支付订单和余额退款。

如果需要理解账户和流水规则，见 [../logic/wallet.md](../logic/wallet.md)。如果需要改代码，见 [../development/wallet.md](../development/wallet.md)。

## 当前边界

当前代码已经支持：

- 后台调整用户余额。
- 前台余额信息和余额流水查询。
- 普通订单使用余额支付。
- 余额支付订单售后退款退回余额。
- 充值套餐后台配置和前台展示。

当前代码尚未形成完整闭环：

- 充值下单。
- 充值支付回调。
- 充值成功自动入账。

因此，充值套餐当前主要用于前台展示和后续充值能力接入准备。真实增加余额时，请使用后台余额调整或已接入的退款链路。

## 后台入口

| 功能 | 入口 | 用途 |
|------|------|------|
| 支付配置 | 设置中心的支付配置页 | 开启或关闭余额支付、微信支付 |
| 充值套餐 | `营销 -> 充值套餐` | 配置前台展示的充值金额、赠送金额和到账余额 |
| 用户余额 | `用户 -> 用户管理` | 查看余额记录、后台调整用户余额 |
| 订单管理 | 订单详情页 | 查看余额支付订单和支付状态 |
| 售后管理 | 售后单处理页 | 余额支付订单退款时退回余额 |

## 前台入口

| 页面 | 路径 |
|------|------|
| 我的余额 | `/pages-sub/wallet/index` |

## 配置前准备

配置前先确认：

1. 普通订单下单和支付流程可以正常使用。
2. 测试用户已经登录。
3. 如需测试余额支付，用户可用余额大于订单应付金额。
4. 如需测试余额退款，售后流程可以进入退款完成。

## 推荐配置流程

### 1. 开启余额支付

进入支付配置页，确认：

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| 余额支付状态 | 开启 | 开启后客户端订单支付方式中可出现余额支付 |
| 微信支付状态 | 按业务需要 | 充值套餐页面当前只展示可用充值方式，微信充值闭环仍需继续接入 |

注意：订单支付接口会再次校验余额支付开关，前端展示不能替代后端准入。

### 2. 给测试用户增加余额

进入 `用户 -> 用户管理`，选择用户后调整余额。

| 字段 | 说明 |
|------|------|
| 调整方向 | 增加或扣减 |
| 调整金额 | 金额单位为元，最多两位小数 |
| 调整原因 | 必填，用于客服和财务排查 |

调整限制：

- 调整金额必须大于 0。
- 单次调整金额不能超过 `999999.99` 元。
- 扣减时用户可用余额必须充足。
- 调整后会写余额流水 `admin_adjust`。

不要直接修改数据库中的 `mb_user_wallet.balance_cents`。

### 3. 配置充值套餐

进入 `营销 -> 充值套餐`，配置前台充值页展示内容。

| 字段 | 说明 |
|------|------|
| 套餐名称 | 前台展示名称 |
| 支付金额 | 用户需要支付的金额 |
| 赠送金额 | 额外赠送金额，可为 0 |
| 到账余额 | 系统自动按支付金额 + 赠送金额计算 |
| 背景图 | 前台套餐卡片展示图 |
| 排序 | 数字越小越靠前 |
| 状态 | 禁用后前台不展示 |
| 备注 | 内部说明 |

运营注意：当前充值套餐不代表充值已自动入账。上线充值闭环前，不要把充值套餐作为真实收款完成凭证。

### 4. 验证余额支付

1. 开启余额支付。
2. 给测试用户增加余额。
3. 用测试用户创建普通订单。
4. 在订单支付页选择余额支付。
5. 支付成功后查看订单状态，应变为已支付。
6. 查看用户余额，应减少订单应付金额。
7. 查看余额流水，应出现 `order_pay`。

### 5. 验证余额退款

1. 用余额支付完成一笔订单。
2. 发起售后退款。
3. 后台审核并执行退款。
4. 查看用户余额，应增加退款金额。
5. 查看余额流水，应出现 `refund`。

## 常见问题

### 支付时看不到余额支付

优先检查：

1. 支付配置中余额支付是否开启。
2. 客户端支付方式接口是否返回余额支付。
3. 当前订单是否仍处于待支付状态。
4. 用户是否已登录。

### 余额支付失败提示余额不足

优先检查：

1. 用户可用余额是否大于订单应付金额。
2. 是否刚发生退款、后台扣减或其它余额支出。
3. 后台看到的金额单位是元，数据库底层是分，排查时注意单位。

### 后台无法扣减余额

可能原因：

- 用户余额不足。
- 调整金额格式不合法。
- 未填写调整原因。
- 管理员身份无效或没有对应权限。

### 配置了充值套餐但充值没有到账

这是当前能力边界。充值套餐已经用于前台展示，但充值下单、支付回调和自动入账还需要继续接入。当前不要把充值套餐配置当成自动入账功能。

### 余额流水和账户余额对不上

优先检查：

1. 是否有人直接改过 `mb_user_wallet` 聚合表。
2. 相关业务动作是否写入了 `mb_user_wallet_log`。
3. 退款是否只对余额支付订单执行余额退回。
4. 排查时统一按分和元换算。
