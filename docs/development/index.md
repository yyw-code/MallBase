## 常用定位

| 要改什么 | 先看 |
|----------|------|
| 后台配置项 | schema / upgrade SQL，再看对应开发文档 |
| 订单金额计算 | 会员、积分、余额开发文档和订单服务 |
| 用户账户变动 | 对应账户 Service 和流水模型 |
| 前台入口显示隐藏 | 客户端装修开发文档和客户端配置接口 |

## 开发约束

- 后端遵循 `Controller -> Service -> Model` 三层结构。
- Swoole 长驻内存下，Service 不保存请求级状态。
- 涉及账户、库存、订单状态的逻辑先校验再事务。
- 列表接口保持 `list` 和 `total` 查询条件同源，并返回 `compact('total', 'list')`。
- 后台路由和权限字段遵循 `System*` 命名、`/:id` 路径参数和后端驱动路由规范。
